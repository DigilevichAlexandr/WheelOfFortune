# Колесо фортуны (.NET 8)

Веб-приложение для розыгрыша призов:
- Регистрация и вход (cookie).
- Участники добавляют очки, формируют доли колеса.
- При вращении случайно выбирается победитель, забирает банк и может списывать очки на призы.
- PWA: можно установить на устройство.

## Локальный запуск

Требования: .NET SDK 8.0+

```bash
dotnet run
```

Откройте `http://localhost:5000` (или порт из вывода в консоли).

## Публикация (Render.com)

1. Залейте проект в **GitHub** (публичный или приватный репозиторий).
2. Зайдите на [render.com](https://render.com) и войдите через GitHub.
3. **New +** → **Blueprint**.
4. Подключите репозиторий с этим проектом.
5. Render подхватит `render.yaml` и создаст веб-сервис из Dockerfile.
6. После деплоя приложение будет доступно по адресу вида `https://wheel-of-fortune-xxx.onrender.com`.

**Важно:** на бесплатном плане Render база SQLite в контейнере сбрасывается при каждом деплое; пользователи и данные не сохраняются между обновлениями. Для постоянного хранения нужен платный диск (Persistent Disk) или внешняя БД.

## API (требуется авторизация, кроме указанных)

- `GET /api/me` — текущий пользователь (без авторизации).
- `POST /api/register` — регистрация: `{ "username": "...", "password": "..." }`.
- `POST /api/login` — вход: `{ "username": "...", "password": "..." }`.
- `POST /api/logout` — выход.
- `GET /api/state` — состояние игры.
- `POST /api/participants` — добавить участника: `{ "name": "Иван", "points": 120 }`.
- `POST /api/spin` — крутить колесо.
- `POST /api/redeem` — списать очки: `{ "winnerId": "...", "prizeCost": 50 }`.
